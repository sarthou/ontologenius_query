Use Ontologenius Query

Ontologenius_query can be used in a multi-ontological use or for a single ontology use.

Single ontology

Launch ontologenius first with one of the available launch files or with an overloaded launch file.

roslaunch ontologenius ontologenius.launch

Then run the ontologenius_query executable for single ontology.

rosrun ontologenius_query ontologenius_query

A new rosservice will apear under the name: "ontologenius_query/fullQuery"

In the single mode, the parameter 'ns' is unused. Just write your query in the parameter 'query'.

Multi ontology

Launch ontologenius first with one of the available launch files or with an overloaded launch file using ontologenius_multi.

roslaunch ontologenius ontologenius_multi.launch

Then run the ontologenius_query executable for multi-ontology.

rosrun ontologenius_query ontologenius_query_multi

A new rosservice will apear under the name: "ontologenius_query/fullQuery"

In the multi mode, put the name of the targetted ontology in the parameter 'ns' and write your query in the parameter 'query'.

Query Examples

Bedroom isA ?type

results:

-

names: [type]

values: [Room]

Give to class of Bedroom.

?table isA Table

results:

-

names: [table]

values: [Table2]

-

names: [table]

values: [Table1]

Give all the tables.

Table1 isIn ?room

results:

-

names: [room]

values: [Bedroom]

Give all the individuals for which the individual 'Table1' is in it.

?object isA Table, ?object isIn ?room

results:

-

names: [object, room]

values: [Table2, Bedroom]

-

names: [object, room]

values: [Table1, Bedroom]

Give all the individuals that are tables and the room in which they are.

?color isA Color, ?color hasLabel ?label

results:

-

names: [color, label]

values: [Red, red]

-

names: [color, label]

values: [Blue, blue]

Give all the colors and their labels.
